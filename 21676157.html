Qeo Open Source Project Documentation : /realms/{id}/users (GET)

Created by lissensj on May 21, 2014
Description: Retrieve all users within a realm of a Qeo administrator 
URL Structure: https://my.qeo.org/qeo-rest-service/v1/realms/{id}/users 
HTTP Method: GET
Parameters:

HTTP Authorization

Path parameter:
    identifier of the realm to be interrogated (value is of type long)

Request Example
https://my.qeo.org/qeo-rest-service/v1/realms/103/users
Response:

HTTP 200 OK
Content-Type: application/json

Body: Contains a list of users found in the realm in JSON format
          {"users":[{"id":...,"realm":...,"type":"user","name":"..."},{...},{...}]}

          The returned parameters per user in the list: 
              "id"       : value of type long, representing the user identifier
              "type"   : type of object returned (always "user")
              "realm" : value of type long, represents the realm identifier
              "name" : UTF-8 string representing the name of the user in the realm

 

Response Example
{
   "users":[{
              "id":3,
              "realm":103,
              "type":"user",
              "name":"Martin"
            },{
              "id":4,
              "realm":103,
              "type":"user",
              "name":"Benny"
            },{
              "id":5,
              "realm":103,
              "type":"user",
              "name":"Sandra"
            }]
}
