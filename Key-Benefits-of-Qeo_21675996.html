Qeo Open Source Project Documentation : Key Benefits of Qeo

Created by lissensj on May 21, 2014

Cross-Platform Standard

Currently, many home automation and Internet of Things solutions rely on platform-related and vendor-related solutions, making integration a painful if not impossible affair. Qeo provides cross-platform standardization, allowing it to be deployed on any device and any operation system.

Distributed Solution

Qeo offers a distributed solution with minimal reliance on a central server. Once your devices are set up and authenticated, they can function independently from the Management Server.

Works Outside the Constraints of the Home

Qeo is not limited to functioning within the home LAN, but functions anywhere in thanks to the Qeo Forwarder which ensures Qeo operation as long as an Internet connection is available.

Easy Set-Up and High Transparency

Qeo is easy to set up and integrates seamlessly with most development environments. It supports a range of common programming languages and platforms. Thanks to its transparent design, applications can be developed on a fairly high level, without the need to know or understand the inner workings. 

Re-Use and Standardization of Components

Qeo is set up in such a way that components can be easily re-used. For example, data structures developed in the Qeo Data Model (QDM) can be translated into components for different programming languages and components using the Code Generator. Data types can be standardized and re-used to ensure inter-operability of applications by different developers.
